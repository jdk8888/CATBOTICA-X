## Source of Truth Protection

### Immutable Core Canon
**These elements CANNOT be modified by community:**

- [x] **Core Event:** The Luck-Module Recalibration Protocol (LMRP) is a mandatory firmware maintenance event, NOT a holiday celebration
- [x] **World Rule:** The full Zodiac Vault (12 templates) was pre-loaded into firmware at Catbotica Industries founding — no zodiac was "discovered" later
- [x] **Visual Rule:** All badges must use the hybrid color palette — Cyan (#00F2FF) circuitry persistent frame + Red (#C41E3A) / Gold (#FFD700) Lunar overlay + Carbon Fiber Black (#0D0D0D) base
- [x] **Material Rule:** All zodiac creatures are rendered in hard-surface mechanical style — plates, wiring, circuit traces, LED accents. No organic textures (fur, skin, feathers)
- [x] **Composition Rule:** Retro-futuristic enamel pin aesthetic with UE5-quality volumetric lighting. 1:1 square format, centered, symmetrical
- [x] **Luck-Module Manifestation:** Every badge features a visible Luck-Module glow in the central forehead region — Cyan when dormant, Gold when actively recalibrating
- [x] **Timeline Anchor:** Collection launched January 2022 (Year of the Tiger). One zodiac activates per Lunar New Year. 2026 = Horse (current)

**Rationale:** These elements define the visual identity and lore integrity of the entire 12-badge collection. Changing any one would break cross-badge consistency and narrative coherence with the LMRP protocol established in the Project Bible.

### Protected Relationships
- **Zodiac Badges ↔ LMRP Protocol:** Badges are proof-of-recalibration issued during LMRP. This relationship cannot be altered.
- **Zodiac Badges ↔ Elementals System:** LIAHONA powers the Luck-Module, AURUM enhances during recalibration, VELOCITITE resonates with Horse frequency. These links are canon.
- **Zodiac Badges ↔ Hold-Duration System:** Retroactive access based on mint date vs. collection launch (Jan 2022). Early minters accumulate larger collections. This mechanic is immutable.
